# Form Tab Ordering

## Intent

Ensure all forms have a logical, predictable tab order that follows visual layout and enables efficient keyboard navigation. This improves accessibility and power-user workflows.

## Specification

All CRUD forms (and custom forms) should have explicit `tabindex` attributes or natural DOM ordering that produces:
1. Tab order follows visual reading order (top-to-bottom, left-to-right)
2. Required fields before optional fields within logical groupings
3. Action buttons (Save, Cancel) at the end of the tab sequence
4. Skip links or logical groupings for complex forms

## Validation

- [ ] Manual keyboard navigation test on each form type
- [ ] Screen reader verification (optional but recommended)
- [ ] Document any forms requiring custom tabindex handling

## Scope

- CRUD forms generated by `crud/routes`
- Custom forms (meditation timer, etc.)
- Does not include: modal focus trapping (separate concern)

## Context

- Forms are rendered in `src/tech/jgood/gleanmo/crud/forms/`
- Input rendering centralized in `src/tech/jgood/gleanmo/crud/forms/inputs.clj`
- Current forms may rely on default DOM order which may not match visual layout
